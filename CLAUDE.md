## Commands

```bash
npm run dev      # Start dev server on port 3000 (Vite, HMR-enabled)
npm run build    # Production build to /dist
npm run lint     # TypeScript type-check only (tsc --noEmit)
npm run preview  # Preview production build locally
```

No test runner is configured. There is no backend — the app is purely client-side.

## Architecture

**The Thaal** is a React 19 + Vite SPA for orchestrating traditional Bohra culinary events. It is deployed on Google AI Studio (Cloud Run), which injects `GEMINI_API_KEY` and `APP_URL` at runtime via environment variables.

### Routing

There is no React Router. Navigation is state-based: `App.tsx` holds a set of boolean flags (`isThaalPlannerOpen`, `isSpicesPageOpen`, etc.) and a `closeAllPages()` helper that resets all of them before activating a new view. The `Navigation` component receives callbacks and fires state changes in `App`.

### State management

All major state lives in `App.tsx` (lifted state, props-drilled down). Local component state handles UI concerns like search, filters, and expanded items. `archivedRecipes` and `archivedPlans` are persisted to `localStorage` via `useEffect`.

### AI integration

Two Gemini models are used inline in components (not in a centralized service):
- `gemini-1.5-flash` — fast recipe generation in `RecipesPage.tsx`
- `gemini-2.5-pro-preview` — ritual/tradition reasoning in `ThaalPlanner.tsx`

The SDK is instantiated lazily inside components: `new GoogleGenAI({ apiKey: import.meta.env.VITE_GEMINI_API_KEY })`. AI errors surface as themed fallback strings ("The Master remains silent") rather than thrown exceptions.

### Styling

Tailwind CSS 4 (via Vite plugin). Custom design tokens are defined in `src/index.css`:
- Colors: `brand-bg` (#0F0D0B), `brand-gold` (#DAA520), `brand-cream` (#E8E1D9)
- Fonts: Cormorant Garamond (serif headings) + Inter (body)
- Utilities: `.glass-card`, `.glass-nav`, `.immersive-gradient`, `.animate-slow-pan`
- Print styles for recipe and logistics output are included in `index.css`

Animations use the `motion` library (Framer Motion API) with `AnimatePresence` for page/modal transitions.

### Key files

| File | Role |
|---|---|
| `src/App.tsx` | Root: all page-level state, navigation logic, localStorage persistence |
| `src/types.ts` | TypeScript interfaces — `Recipe` and `ThaalPlan` (the two core data models) |
| `src/constants.ts` | Flavor profiles, cuisine types, common ingredients |
| `src/data/verifiedRecipes.ts` | Hardcoded verified recipe records |
| `src/components/ThaalPlanner.tsx` | ~900 lines — the event orchestration interface, multi-step planner, AI audit generation |
| `src/components/RecipesPage.tsx` | Recipe gallery + AI recipe synthesis from pantry ingredients |
| `src/components/KitchenMode.tsx` | Full-screen step-by-step recipe execution |

### Data models

`ThaalPlan` is the dominant complex type (~190 fields). It contains a `dishes[]` array, a synchronized `timeline[]` across three tracks (Active Cooking / Host Rituals / Prep+Plating), a scaled `masterIngredients[]` list, and multiple named audit sections (`engineeringAudit`, `qaAudit`, `securityAudit`, `safetyAudit`, `documentationAudit`, `combinationsAudit`, etc.) — all generated by the Gemini API and stored client-side.

`Recipe` has 18 fields including `category` (Meethas / Kharaas / Jamaan / Salad), `flavorProfile`, `cuisineType`, `heritage` (cultural context string), and `verified` boolean.

### Path alias

`@/*` resolves to the project root (configured in both `tsconfig.json` and `vite.config.ts`).

### AI Studio compatibility

`vite.config.ts` conditionally disables HMR when `DISABLE_HMR=true` is set. Do not remove this logic — it is required for the Cloud Run deployment environment.
